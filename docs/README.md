## 當前設計原則

- 先設計、後實現，避免模型環境尚未就緒時反覆推翻 Web 架構。
- 管理域與算法域解耦，將 Django 作為核心業務後台，FastAPI 作為推理能力服務。
- 前端僅對接統一業務入口，避免直接耦合模型服務細節。
- 高內聚低耦合，優先明確模組邊界、數據契約與可替換點。
- 以可演進架構為目標，先支持本地開發與單機部署，再平滑演進到異步任務與分布式推理。
